Rocketlog - Histórico do Projeto

Este documento apresenta a ordem cronológica de desenvolvimento do projeto Rocketlog, bem como a lista de todas as dependências instaladas até o momento.

1. Passo a Passo do Projeto (Fluxo de Trabalho)

  1. Inicialização do Projeto: Criação do arquivo base de configuração package.json executando o comando: 
    npm init -y
  2. Estruturação e Configuração Inicial do Express: Instalação do framework de rotas Express e de sua tipagem TypeScript correspondente.
  3. Ambiente de Desenvolvimento TypeScript: Instalação do compilador TypeScript e de executores de desenvolvimento (ts-node, tsx, tsc-alias). Criação do arquivo de configurações tsconfig.json.
  4. Refatoração da Inicialização do Servidor: Separação da lógica de rotas/servidor:
    • src/app.ts: Responsável por instanciar o Express, definir middlewares e exportar a instância.
    • src/server.ts: Responsável por escutar na porta definida (usando process.env.PORT || 3333).
  5. Dockerização e Hot-Reload Automático: Criação de arquivos de configuração Docker:
    • Dockerfile: Estruturado em múltiplos estágios (base, development, build, runner).
    • .dockerignore: Ignorando node_modules e pastas de build.
    • docker-compose.yml: Orquestrando o container da aplicação montado com volume dinâmico local e habilitando a variável CHOKIDAR_USEPOLLING=true para contornar problemas de notificação de arquivos no Windows.
  6. Validação e Tratamento de Erros: Instalação de ferramentas de validação de dados (zod) e captura de erros assíncronos (express-async-errors). Implementação do middleware global de tratamento de erros errorHandler em src/midllewares/error-handling.ts, configurado para tratar erros de validação do Zod, instâncias de AppError e falhas inesperadas (500).
  7. Banco de Dados PostgreSQL no Docker: Configuração e inicialização do serviço do banco de dados PostgreSQL (imagem postgres:16-alpine) exposto na porta 5432 com volume persistente (pgdata).
  8. Configuração e Inicialização do ORM Prisma: Instalação do CLI do Prisma e inicialização da estrutura rodando o comando diretamente de dentro do container do Docker (solucionando o conflito com a versão local v24 do Node.js): 
    docker compose exec app npx prisma init --datasource-provider postgresql

2. Bibliotecas Instaladas (Ordem Cronológica)

A. Dependências de Produção (dependencies)

  • express (v4.19.2) - Framework para controle de rotas e requisições HTTP.
  • express-async-errors (v3.1.1) - Middleware para tratamento automático de erros assíncronos no Express.
  • zod (v3.23.8) - Biblioteca de validação de dados e construção de schemas.

B. Dependências de Desenvolvimento (devDependencies)

  • @types/express (v4.17.21) - Tipagem para TypeScript do framework Express.
  • typescript (v5.5.4) - Linguagem e compilador TypeScript.
  • @types/node (v20.14.12) - Tipagem para as APIs padrão do Node.js.
  • ts-node (v10.9.2) - Executor TypeScript direto no ambiente de desenvolvimento.
  • tsx (v4.16.2) - Ferramenta rápida para executar arquivos .ts com modo de escuta automática (watch).
  • tsc-alias (v1.8.17) - Utilitário para resolver os caminhos com alias (ex: @/*) na pasta de distribuição (build).
  • prisma (v5.19.1) - Interface de CLI para o ORM Prisma.
